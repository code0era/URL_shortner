# 💾 SnipURL — Database & Schema Documentation

> MongoDB database design using Mongoose ODM.

## Database Overview

- **Database Name**: `urlshortener`
- **Collection**: `urls`
- **ODM**: Mongoose 8.x
- **Pattern**: Embedded sub-documents for click tracking (denormalized)

## Schema Diagram

```
┌─────────────────────────────────────────────────────────┐
│                     urls (Collection)                    │
├─────────────────────────────────────────────────────────┤
│  _id            : ObjectId (auto, PK)                   │
│  originalUrl    : String (required, trimmed)             │
│  shortId        : String (required, unique, indexed)     │
│  customAlias    : String (unique, sparse, trimmed)       │
│  clicks         : Number (default: 0)                    │
│  isActive       : Boolean (default: true)                │
│  expiresAt      : Date (default: null)                   │
│  password       : String (default: null)                 │
│  maxClicks      : Number (default: null)                 │
│  tags           : [String]                               │
│  createdBy      : String (default: "anonymous")          │
│  createdAt      : Date (auto, via timestamps)            │
│  updatedAt      : Date (auto, via timestamps)            │
│                                                         │
│  clickDetails   : [ClickSchema]  ◄── Embedded array     │
│    ├── timestamp : Date (default: now)                   │
│    ├── referrer  : String (default: "Direct")            │
│    ├── userAgent : String (default: "Unknown")           │
│    └── ipAddress : String (default: "Unknown")           │
└─────────────────────────────────────────────────────────┘
```

## Field Descriptions

### Core Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `originalUrl` | String | ✅ | The long URL to redirect to |
| `shortId` | String | ✅ | 8-char ID generated by nanoid |
| `customAlias` | String | ❌ | User-defined custom short code |
| `clicks` | Number | ❌ | Total click counter (denormalized) |
| `isActive` | Boolean | ❌ | Toggle to enable/disable URL |
| `expiresAt` | Date | ❌ | Auto-expire after this timestamp |
| `password` | String | ❌ | Optional password protection |
| `maxClicks` | Number | ❌ | Stop redirecting after N clicks |
| `tags` | [String] | ❌ | User-defined categorization tags |
| `createdBy` | String | ❌ | Creator identifier |

### Click Tracking (Embedded Sub-document)

| Field | Type | Description |
|-------|------|-------------|
| `timestamp` | Date | When the click happened |
| `referrer` | String | HTTP Referrer header |
| `userAgent` | String | Browser/client User-Agent |
| `ipAddress` | String | Client IP address |

## Indexes

```javascript
shortId     : { unique: true }     // Primary lookup — O(log n)
createdAt   : { descending: -1 }   // Sort by newest first
clicks      : { descending: -1 }   // Sort by most popular
customAlias : { unique, sparse }   // Only indexed when present
```

## Virtual Fields

```javascript
isExpired       → true if current date > expiresAt
maxClicksReached → true if clicks >= maxClicks
```

Both virtuals are included in JSON/Object output via `{ virtuals: true }`.

## Why Embedded Documents?

**Pros of embedding `clickDetails` inside `urls`:**
- No JOINs needed — single query gets URL + all clicks
- Atomic updates — click recording is part of URL update
- Simpler schema — one collection to manage

**Trade-offs:**
- MongoDB document size limit is 16MB
- For viral URLs with millions of clicks, a separate `clicks` collection would be needed
- For this project's scale, embedding is optimal

## Aggregation Pipelines Used

### Total Clicks Across All URLs
```javascript
Url.aggregate([{ $group: { _id: null, total: { $sum: '$clicks' } } }])
```

### Clicks Per Day (Last 7 Days)
```javascript
Url.aggregate([
  { $unwind: '$clickDetails' },
  { $match: { 'clickDetails.timestamp': { $gte: sevenDaysAgo } } },
  { $group: { _id: { $dateToString: { format: '%Y-%m-%d', date: '$clickDetails.timestamp' } }, count: { $sum: 1 } } },
  { $sort: { _id: 1 } }
])
```
